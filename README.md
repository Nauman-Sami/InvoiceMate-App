# Invoice App — Setup Guide

Flutter + GetX + Hive (offline) + Firebase (cloud sync)

---

## Prerequisites

- Flutter SDK ≥ 3.0.0 installed
- Android Studio or VS Code
- A Google account (for Firebase)
- Node.js installed (needed for Firebase CLI)

---

## Step 1 — Get Flutter Dependencies

```bash
cd invoice_app
flutter pub get
```

---

## Step 2 — Firebase Project Setup

### 2a. Create Firebase Project
1. Go to https://console.firebase.google.com
2. Click **Add project** → name it `invoice-app` → Continue
3. Disable Google Analytics (optional) → **Create project**

### 2b. Enable Authentication
1. In Firebase Console → **Authentication** → **Get started**
2. Enable **Email/Password** provider
3. Enable **Google** provider (add your support email)

### 2c. Create Firestore Database
1. In Firebase Console → **Firestore Database** → **Create database**
2. Choose **Start in test mode** (for development)
3. Select a region close to you (e.g. `asia-south1` for Pakistan)
4. Click **Enable**

### 2d. Firestore Security Rules (paste this in Rules tab)
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

---

## Step 3 — Connect Firebase to Flutter (FlutterFire CLI)

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Login to Firebase
firebase login

# In your project root, run:
flutterfire configure
```

- Select your `invoice-app` Firebase project
- Select **Android** (and iOS if needed)
- This auto-generates `lib/firebase_options.dart` with real values ✅

---

## Step 4 — Add google-services.json (Android)

After running `flutterfire configure`, a `google-services.json` is placed at:
```
android/app/google-services.json
```

If not placed automatically, download it from:
Firebase Console → Project Settings → Your apps → Android app → Download `google-services.json`
Place it at `android/app/google-services.json`

---

## Step 5 — Android Gradle Config

Make sure `android/build.gradle` has:
```gradle
buildscript {
    dependencies {
        classpath 'com.google.gms:google-services:4.4.0'
    }
}
```

And `android/app/build.gradle` has at the bottom:
```gradle
apply plugin: 'com.google.gms.google-services'
```

Also set `minSdkVersion` to at least **21**:
```gradle
android {
    defaultConfig {
        minSdkVersion 21
        targetSdkVersion 34
    }
}
```

---

## Step 6 — Run the App

```bash
flutter run
```

## Firestore Data Structure

```
users/
  {userId}/
    clients/
      {clientId}  → ClientModel fields
    products/
      {productId} → ProductModel fields
    invoices/
      {invoiceId} → InvoiceModel fields (items as array)
```

---

## Notes

- **Offline**: All data is saved to Hive first. Firebase sync happens in background.
- **Sync**: On login, pulls latest from Firestore. On any save, pushes pending records.
- **PDF**: Generated fully on-device, no server needed.
- **Free tier**: Firebase Spark plan is free. 20k writes/day, 50k reads/day — more than enough for personal/small business use.
